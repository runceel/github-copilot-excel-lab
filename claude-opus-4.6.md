## 1. 本ドキュメントについて

この仕様書は、従業員管理システムのデータ項目定義を記載しています。

## 2. 従業員データ

従業員データは以下のような構造を持ったデータで単一テーブルにて表現されます。

| 項番 | 項目名 | 型 | 備考 |
|------|--------|--------|------|
| 1 | ID | int | 従業員の識別子。1 からはじまる連番 |
| 2 | EmployeeID | string | 従業員番号。従業員を識別するための一意な番号。入社年度と連番で構成される。 |
| 3 | FirstName | string | 名前 |
| 4 | LastName | string | 苗字 |
| 5 | BaseSalary | int | 月の基本給 |
| 6 | DepartmentName | string | 所属部署名 |
| 7 | BusinessFirstName | string | ビジネス用の名前 |
| 8 | BusinessLastName | string | ビジネス用の苗字 |
| 9 | Memo | string | 特記事項などを記載する項目。 |

### 特記事項 (2026/02/16 追記)

従業員の Memo 列には JSON 形式の文字列により以下の項目を格納する。

Memo 列は歴史的経緯から追加プロパティを入力する運用になっているが、初期バージョンからある画面では JSON 形式ではない文字列を格納している部分もあるため、Memo 列を使用する場合はその点に考慮すること。

## 3. 設計上の決定事項

本来は以下のような構造であるべきですが、本システムでは単純化のためこのような構造になっています。

```mermaid

    Employees["Employees"] -->|参照| Departments["Departments"]
```

## 4. 本方式のメリット

単純な構造を保ちつつ、柔軟な列追加が可能な JSON を組み合わせることで複雑な構造に拡張も可能になっています。
